Letter.Doc

  • Record

Represents a logical unit or node within the documentation structure, typically corresponding to a file (e.g., Markdown) or a conceptual section.

A documentation node includes a display title, a reference path (which might be a file path, URL segment, or identifier), and potentially a list of nested sub-nodes (children) to represent hierarchical relationships (e.g., chapters and sections).

Doc(Stringtitle, Stringpath)

Stringtitle

the title of the node.
Stringpath

the path associated with the node.

Convenience constructor to create a new documentation node (a leaf node) without any sub-nodes. Initializes the sub-node list as an empty list.
