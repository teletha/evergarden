Letter

Represents the central metadata and configuration for a documentation project.

This abstract class serves as a container for various pieces of information gathered during the documentation generation process, such as discovered modules, packages, parsed class information, and the structure of external documentation files.

Subclasses must implement the abstract methods to provide project-specific details like the project title, root directory, character encoding, hosting information, and logic for retrieving code samples.

modulesListString

The list of JPMS (Java Platform Module System) module names declared in the project. This list is populated during the scanning phase.

packagesListString

The list of Java package names discovered within the project's source code during scanning.

typesListClassInfo

A collection of ClassInfo objects representing type definitions (classes, interfaces, enums, records) parsed from the project's Java source files.

docsListDoc

A tree structure representing external documentation files (e.g., Markdown files) and their logical hierarchy, defined using Doc nodes. This typically forms the main navigation structure of the generated documentation site.
